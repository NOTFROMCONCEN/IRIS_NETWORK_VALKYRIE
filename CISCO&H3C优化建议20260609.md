## 一、严重问题（P0）

### 1.1 CiscoAdapter 设备类型错误

**涉及文件：** `CoreBase/core/adapters.py` 第 812-833 行

**当前代码：**
```python
class CiscoAdapter(BaseAdapter):
    def __init__(self, ...):
        ...
        self.device_type = "cisco_ios"    # ← 第 832 行
```

**问题描述：**
项目中实际管理的 Cisco 设备为 MDS 9710 和 MDS 9148S 光纤交换机，这些设备运行 NX-OS 操作系统，而非 IOS。`cisco_ios` 是 netmiko 用于 Cisco IOS/IOS-XE 设备的 device_type，与 NX-OS 设备的命令语法、提示符格式、分页机制完全不同。

**实际影响：**
- netmiko 使用 `cisco_ios` 模式时会按 IOS 提示符（如 `Router#`）匹配，而 MDS 设备提示符格式为 `switch#` 或 `SD97SN0A-N1#`，可能导致提示符匹配失败
- NX-OS 的分页关闭命令是 `terminal length 0`，IOS 用的是 `terminal length 0`（相同）但交互行为不同
- 部分命令语法差异可能导致输出截断或命令执行失败
- 实际巡检中 4 台 MDS 设备全部通过，说明 netmiko 在 `cisco_ios` 模式下有一定的兼容性，但这属于"碰巧能工作"，不可靠

**修正方案：**

方案 A — 基于设备名自动判断（推荐）：
```python
class CiscoAdapter(BaseAdapter):
    def __init__(self, ...):
        ...
        # 根据设备名前缀判断设备类型
        name = (device_name or "").upper()
        if "SN" in name or "MDS" in name or "Nexus" in name:
            self.device_type = "cisco_nxos"
        else:
            self.device_type = "cisco_ios"
```

方案 B — 在 commands.yaml 中按主机名配置：
```yaml
hostnames:
  SD97SN0A-N1:
    vendor: cisco
    device_type: cisco_nxos
    standard:
      version: show version
      ...
```
此方案需要在 `BaseAdapter.__init__` 中增加 `device_type` 的主机名覆盖逻辑。

方案 C — 在 devices.xlsx 中增加 device_type 列：
让用户在设备清单中直接指定每台设备的 netmiko device_type，适配器从配置读取。

### 1.2 H3C 缺少分页关闭预处理

**涉及文件：** `CoreBase/core/adapters.py` 第 700-725 行（`H3CAdapter`）

**当前代码：**
`H3CAdapter` 直接使用 `hp_comware` device_type 连接并执行命令，没有在连接后、命令执行前发送分页关闭命令。

**问题描述：**
H3C Comware 设备默认开启分页显示（每屏24行），超过一屏的命令输出会出现 `---- More ----` 提示符，需要按空格或回车继续。实际巡检中所有设备的日志都先执行了 `screen-leng dis`（v5）或 `screen-length disable`（v7）来关闭分页。

**实际影响：**
- `dis cpu-usage`、`dis memory`、`dis int brief`、`dis logbuffer reverse` 等命令输出可能超过24行
- 如果未关闭分页，netmiko 的 `send_command` 会因遇到 `---- More ----` 而超时或返回截断的输出
- 虽然部分 netmiko 版本会自动发送空格键处理分页，但这不可靠，且会导致输出中混入 `---- More ----` 字符，干扰后续解析

**修正方案：**

在 `H3CAdapter` 中覆盖 `connect()` 方法，连接成功后自动发送分页关闭命令：
```python
class H3CAdapter(BaseAdapter):
    def __init__(self, ...):
        ...
        self.device_type = "hp_comware"

    def connect(self) -> Tuple[bool, str]:
        success, message = super().connect()
        if success:
            # 连接成功后关闭分页
            try:
                self.send_command("screen-length disable", read_timeout=10)
            except Exception:
                # v5 设备可能使用不同命令，忽略错误
                try:
                    self.send_command("screen-leng dis", read_timeout=10)
                except Exception:
                    pass
        return success, message
```

### 1.3 commands.yaml 中 Cisco 命令集不适用于 MDS 设备

**涉及文件：** `CoreBase/config/commands.yaml` 第 107-121 行

**当前配置：**
```yaml
  cisco:
    standard:
      version: show version
      running_config: show running-config
      inventory: show inventory
      processes: show processes cpu
      memory: show memory statistics
      mac: show mac address-table
      vlan: show vlan brief
      bgp: show ip bgp summary
      ospf: show ip ospf neighbor
    logs:
      logging: show logging
```

**问题描述：**
以上命令是标准 Cisco IOS 路由器/交换机的命令集，而实际设备为 MDS 光纤交换机（NX-OS），两者差异很大：

| 检查项 | 当前命令（IOS） | 实际需要的命令（NX-OS MDS） | 状态 |
|--------|----------------|---------------------------|------|
| 电源状态 | 缺失 | `show environment power` | 缺失 |
| 风扇状态 | 缺失 | `show environment fan` | 缺失 |
| 温度状态 | 缺失 | `show environment temperature` | 缺失 |
| CPU利用率 | `show processes cpu` | `show processes cpu` | 可用 |
| 内存利用率 | `show memory statistics` | `show system resources` | 需要更换 |
| 端口状态 | 缺失 | `show flogi database` / `show interface brief` | 缺失 |
| VLAN | `show vlan brief` | MDS 无 VLAN 概念 | 不适用 |
| BGP | `show ip bgp summary` | MDS 不运行 BGP | 不适用 |
| OSPF | `show ip ospf neighbor` | MDS 不运行 OSPF | 不适用 |
| 系统版本 | `show version` | `show version` | 可用 |
| 硬件信息 | `show inventory` | `show inventory` | 可用 |
| 运行配置 | `show running-config` | `show running-config` | 可用（但输出很长） |

**实际影响：**
- 3 个命令（vlan、bgp、ospf）在 MDS 上执行会返回错误或空输出
- 电源、风扇、温度 3 个关键检查项完全缺失
- 内存命令 `show memory statistics` 在 MDS 上输出格式不同，无法直接使用

**修正方案：**

使用 commands.yaml 的 hostnames 功能为 MDS 设备配置专用命令集（参见 [4.2 节](#42-cisco-命令修正)），或在 `vendors.cisco` 中添加 `show environment` 系列命令并使用 fallback_commands 兼容两种设备。

## 二、功能缺失（P1）

### 2.1 零输出解析 — 无法生成结构化巡检报告

**涉及文件：**
- `CoreBase/core/engine.py` 第 351-395 行（`_test_standard_mode`）
- `CoreBase/core/saver.py` 第 35-67 行（`save_result`）

**当前行为：**
```python
# engine.py 第 386-393 行
if result_saver:
    for cmd_type, result in results.items():
        if result.get("status"):
            output = result.get("output", "")
            result_saver.save_result(device_name, ip, cmd_type, output, vendor)
```

命令输出作为原始文本字符串直接写入 `.log` 文件，没有任何解析或结构化处理。

**问题描述：**
巡检的核心价值不是"收集日志"，而是"从日志中提取关键指标并判断设备健康状态"。当前项目只完成了第一步（收集），完全缺少第二步（解析和分析）。

**实际影响：**
- 收集到的日志需要人工逐台阅读和提取数据
- 无法自动生成类似以下巡检报告中的结构化表格：
  ```
  | 设备名称 | IP | CPU 5分钟 | 内存利用率 | 电源状态 | 结论 |
  ```
- 无法自动检测异常（温度告警、CPU 飙高、电源故障等）
- 每月巡检仍然需要大量人工操作，工具只节省了 SSH 连接的时间

**修正方案：**

新增 `CoreBase/core/parsers.py` 模块（详见 [5.1 节](#51-输出解析器-parserspy)），为每个厂商实现命令输出解析器，提取结构化数据。

### 2.2 H3C 不区分 Comware v5/v7 格式

**涉及文件：** `CoreBase/core/adapters.py` 第 700-725 行（`H3CAdapter`）

**当前代码：**
```python
class H3CAdapter(BaseAdapter):
    def __init__(self, ...):
        ...
        self.device_type = "hp_comware"   # 不区分 v5/v7
```

**问题描述：**
H3C Comware v5 和 v7 的命令输出格式存在系统性差异，同一命令的输出结构完全不同：

| 检查项 | Comware v5 格式 | Comware v7 格式 |
|--------|----------------|----------------|
| 电源 | 键值对 `Power 1 / State : Normal` | 表格 `PowerID State Mode ...` |
| 风扇 | 键值对 `FAN 1 / State : Normal` | 表格 `Fan Model Status ...` |
| 温度 | 4列 `SlotNo Temperature Lower Upper` | 7列 `Slot Sensor Temperature Lower Warning Alarm Shutdown` |
| CPU | `Unit CPU usage:` | `Slot X CPU 0 CPU usage:` |
| 内存 | `Used Rate: 26%`（直接给利用率） | `FreeRatio`（需计算 100-FreeRatio） |
| ARP | `Total dynamic entry(ies): X` | `Total number of dynamic entries: X` |

**实际设备分布：**
- v5 设备：SD30RT10-C1（93.1.115.100，MSR30-40）
- v7 设备：其余 16 台（S5130S、S6805、S5590、S12504G、SR6616 等）

**实际影响：**
- 如果用 v7 的正则去解析 v5 的输出，会提取不到数据（返回 None，导致异常）
- 如果用 v5 的正则去解析 v7 的输出，同样会失败
- 当前代码不解析输出所以暂时没暴露问题，但一旦加入解析器就会立即遇到

**修正方案：**

在 `H3CAdapter` 中增加版本自动检测：
```python
class H3CAdapter(BaseAdapter):
    def __init__(self, ...):
        ...
        self.comware_version = None  # 连接后自动检测

    def _detect_comware_version(self, content: str) -> int:
        """从 show version 输出检测 Comware 版本"""
        # v5 特征：Used Rate、Total dynamic entry(ies)
        if 'Used Rate:' in content or 'Comware Software, Version 5' in content:
            return 5
        # v7 特征：FreeRatio、Total number of dynamic entries
        if 'FreeRatio' in content or 'Comware Software, Version 7' in content:
            return 7
        return 7  # 默认 v7（当前主流）
```

解析器根据 `self.comware_version` 选择不同的正则模式。

### 2.4 缺少异常检测与阈值告警机制

**涉及文件：** 全局架构缺失，无对应文件

**问题描述：**
当前项目没有任何阈值判断逻辑。巡检报告需要以下异常检测能力：

| 检查项 | 正常 | 警告 | 严重 |
|--------|------|------|------|
| 温度 | < WarningLimit × 80% | ≥ WarningLimit × 80% | ≥ AlarmLimit |
| CPU 5分钟均值 | < 60% | ≥ 60% | ≥ 80% |
| 内存利用率 | < 80% | ≥ 80% | ≥ 90% |
| 电源状态 | Normal/Ok | — | 非 Normal/Ok |
| 风扇状态 | Normal/Ok | — | 非 Normal/Ok |
| 运行时间 | > 1天 | < 1天（可能异常重启） | — |

**实际影响：**
- 无法自动识别异常设备
- 运维人员需要逐台查看原始日志判断设备状态
- 无法生成带有"结论"列的巡检表格

**修正方案：**

新增 `CoreBase/core/analyzer.py` 模块（详见 [5.2 节](#52-异常分析器-analyzerpy)）。

## 三、架构优化（P2）

### 3.1 不支持多 Slot 设备

**涉及文件：** `CoreBase/core/adapters.py` 全局架构

**问题描述：**
以下设备具有多个独立 Slot，每个 Slot 有自己的 CPU、内存和温度传感器：

| 设备型号 | 设备名称 | IP | Slot 数量 | 说明 |
|----------|---------|-----|-----------|------|
| SR6616 | SD66WA03-A2 | 93.2.131.23 | 2（Slot0 主控、Slot1 备主控） | 路由器 |
| S12504G | SD12SW01-RF2-CSWZ | 93.220.51.1 | 8（业务Slot）+ 2（主控） | 核心交换机 |

当前代码的 `run_commands` 返回的是 `Dict[str, Any]`，以命令类型为键，扁平存储结果，无法表达多 Slot 的层次结构。

**实际影响：**
- `dis cpu-usage` 在多 Slot 设备上返回每个 Slot 的独立 CPU 数据，需要分别提取
- `dis memory` 同理，每个 Slot 有独立的内存统计
- `dis environment` 的温度传感器按 Slot 编号排列
- 如果不按 Slot 分别提取，会丢失部分数据或只保留最后一个 Slot 的值

**修正方案：**

解析器返回数据结构应支持嵌套：
```python
{
    "device": "SD66WA03-A2",
    "ip": "93.2.131.23",
    "cpu": [
        {"slot": 0, "cpu_5s": "11%", "cpu_1m": "9%", "cpu_5m": "9%"},
        {"slot": 1, "cpu_5s": "4%", "cpu_1m": "4%", "cpu_5m": "4%"}
    ],
    "memory": [
        {"slot": 0, "used_pct": 26.4, "conclusion": "正常"},
        {"slot": 1, "used_pct": 25.6, "conclusion": "正常"}
    ]
}
```

### 3.2 ResultSaver 只输出原始日志

**涉及文件：** `CoreBase/core/saver.py` 全文件

**当前行为：**
- `save_result()` — 将命令输出追加写入 `.log` 文件，格式为 `设备名(IP)(日期)(厂商).log`
- `save_summary()` — 生成纯文本汇总报告（只有成功/失败统计）
- `save_failed_devices()` — 生成失败设备列表

**问题描述：**
缺少巡检报告的核心输出能力：
1. 没有 Markdown 格式的结构化报告
2. 没有按检查项（电源/风扇/温度/CPU/内存等）的分类表格
3. 没有异常设备汇总表
4. 没有与历史数据的对比功能

**实际影响：**
- 当前输出无法直接用于提交巡检报告
- 需要人工从 `.log` 文件中提取数据、制作表格
- 无法自动对比"本月 vs 上月"的变化趋势

**修正方案：**

新增 `CoreBase/core/reporter.py` 模块（详见 [5.3 节](#53-报告生成器-reporterpy)），或扩展 `ResultSaver` 增加以下方法：
- `save_inspection_report()` — 按厂商生成 Markdown 巡检报告
- `save_anomaly_summary()` — 生成异常设备汇总
- `save_diff_report()` — 对比历史数据

### 3.3 H3C Container 内存未截断

**涉及文件：** 解析器（待实现）

**问题描述：**
H3C Comware v7 设备的 `display memory` 命令输出包含两段：

```
Memory statistics are measured in KB(Available):
Slot 0:
             Total      Used      Free    Shared   Buffers    Cached   FreeRatio
Mem:      15490384   4529300  10961084         0      1176   1724296       73.6%
...

Container memory statistics are measured in KB(Available):    ← 分界线
Slot 0:
             Total      Used      Free  UsageRatio
Mem:      15490384    847436  10960820        5.5%
...
```

**实际影响：**
- 如果不截断 Container 部分，解析器会提取到两组 Slot0/Slot1 数据
- Container 内存的 FreeRatio/UsageRatio 与主内存不同，混用会导致内存利用率计算错误
- 例如：主内存 FreeRatio 73.6% → 利用率 26.4%，Container UsageRatio 5.5% → 差距巨大

**修正方案：**

解析器在处理 `display memory` 输出时，先截断 Container 部分：
```python
content = content.split('Container')[0]
```

这一处理必须在解析器中实现，不能在命令执行阶段做（因为 netmiko 返回的是完整输出）。

## 四、commands.yaml 具体修正方案

### 4.1 H3C 命令修正

**文件：** `CoreBase/config/commands.yaml` 第 39-53 行

**修正后配置：**
```yaml
  h3c:
    standard_groups:
      - display_l3
    standard:
      version: display version
      device: display device
      power: display power
      fan: display fan
      temperature: display environment
      cpu: display cpu-usage
      memory: display memory
      interface_brief: display interface brief
      arp_dynamic:
        command: display arp dynamic count
        fallback_commands:
          - display arp dyna count
      arp_static:
        command: display arp static count
      mac_count: display mac-address count
      routing_table: display ip routing-table

    logs:

```

### 5.1 输出解析器 parsers.py

**建议位置：** `CoreBase/core/parsers.py`

**设计思路：**
每个厂商一个解析器类，统一接口，按检查项提供解析方法。

```python
"""
设备巡检输出解析器
"""
import re
from abc import ABC, abstractmethod
from typing import Dict, List, Any, Optional
from dataclasses import dataclass


@dataclass
class ParsedResult:
    """解析结果数据类"""
    device_name: str
    ip: str
    vendor: str
    version: Optional[str] = None
    model: Optional[str] = None
    serial: Optional[str] = None
    uptime: Optional[str] = None
    power: List[Dict[str, str]] = None      # [{"module": "Power 1", "state": "Normal"}]
    fan: List[Dict[str, str]] = None         # [{"module": "Fan 1", "state": "Ok"}]
    temperature: List[Dict[str, Any]] = None # [{"sensor": "...", "temp": 30, "alarm": 70, ...}]
    cpu: List[Dict[str, Any]] = None         # [{"slot": 0, "5s": "11%", "1m": "9%", "5m": "9%"}]
    memory: List[Dict[str, Any]] = None      # [{"slot": 0, "used_pct": 26.4}]
    routes: Optional[Dict[str, int]] = None  # {"destinations": 2215, "routes": 2787}
    arp: Optional[Dict[str, int]] = None     # {"dynamic": 360, "static": 0}
    mac_count: Optional[int] = None


class BaseParser(ABC):
    """解析器基类"""

    @abstractmethod
    def parse_all(self, device_name: str, ip: str, outputs: Dict[str, str]) -> ParsedResult:
        """解析所有命令输出"""
        pass


class H3CParser(BaseParser):
    """H3C Comware 输出解析器"""

    def __init__(self, comware_version: int = 7):
        self.version = comware_version

    def parse_all(self, device_name: str, ip: str, outputs: Dict[str, str]) -> ParsedResult:
        result = ParsedResult(device_name=device_name, ip=ip, vendor="H3C")
        for cmd_type, output in outputs.items():
            if cmd_type == "version":
                result.version, result.uptime = self._parse_version(output)
            elif cmd_type == "cpu":
                result.cpu = self._parse_cpu(output)
            elif cmd_type == "memory":
                result.memory = self._parse_memory(output)
            # ... 其他检查项
        return result

    def _parse_cpu(self, content: str) -> List[Dict[str, Any]]:
        """解析 CPU 利用率，支持多 Slot"""
        # v7 多 Slot 格式
        matches = re.findall(
            r'Slot\s+(\d+)\s+CPU\s+(\d+)\s+CPU usage:\s*\n'
            r'\s*(\d+)%\s+in\s+last\s+5\s+seconds\s*\n'
            r'\s*(\d+)%\s+in\s+last\s+1\s+minute\s*\n'
            r'\s*(\d+)%\s+in\s+last\s+5\s+minutes',
            content, re.MULTILINE
        )
        if matches:
            return [
                {"slot": int(m[0]), "cpu_id": int(m[1]),
                 "5s": f"{m[2]}%", "1m": f"{m[3]}%", "5m": f"{m[4]}%"}
                for m in matches
            ]
        # v5 格式 fallback
        match = re.search(r'CPU usage:\s*\n\s*(\d+)%.*?(\d+)%.*?(\d+)%', content, re.DOTALL)
        if match:
            return [{"slot": None, "5s": f"{match.group(1)}%",
                     "1m": f"{match.group(2)}%", "5m": f"{match.group(3)}%"}]
        return []

    def _parse_memory(self, content: str) -> List[Dict[str, Any]]:
        """解析内存利用率，自动截断 Container 部分"""
        content = content.split('Container')[0]  # 关键：截断 Container 内存段

        results = []
        if self.version == 7:
            # v7: 从 FreeRatio 计算
            slots = re.finditer(r'Slot\s+(\d+):(.*?)(?=Slot\s+\d+:|$)',
                                content, re.DOTALL)
            for m in slots:
                slot_id = m.group(1)
                slot_content = m.group(2)
                fr_match = re.search(r'FreeRatio\s*\n[^%]*?(\d+(?:\.\d+)?)%',
                                     slot_content)
                if fr_match:
                    free_ratio = float(fr_match.group(1))
                    used_pct = round(100 - free_ratio, 1)
                    results.append({"slot": int(slot_id), "used_pct": used_pct})
        else:
            # v5: 直接取 Used Rate
            match = re.search(r'Used Rate:\s*(\d+)%', content)
            if match:
                results.append({"slot": None, "used_pct": int(match.group(1))})
        return results

    def _parse_version(self, content: str) -> tuple:
        """解析系统版本和运行时间"""
        ver = re.search(r'Comware Software, Version\s+(.+)', content)
        uptime = re.search(r'uptime is\s+(.*?)(?:\n|$)', content)
        return (ver.group(1).strip() if ver else None,
                uptime.group(1).strip() if uptime else None)


class CiscoParser(BaseParser):
    """Cisco NX-OS (MDS) 输出解析器"""

    def parse_all(self, device_name: str, ip: str, outputs: Dict[str, str]) -> ParsedResult:
        result = ParsedResult(device_name=device_name, ip=ip, vendor="Cisco")
        for cmd_type, output in outputs.items():
            if cmd_type == "version":
                result.version, result.uptime = self._parse_version(output)
            elif cmd_type == "inventory":
                result.model, result.serial = self._parse_inventory(output)
            # ... 其他检查项
        return result

    def _parse_version(self, content: str) -> tuple:
        """解析 NX-OS 版本和运行时间"""
        ver = re.search(r'kickstart:\s+version\s+(\S+)', content) or \
              re.search(r'system:\s+version\s+(\S+)', content)
        uptime = re.search(r'uptime is\s+(.*?)(?:\n|$)', content)
        return (ver.group(1) if ver else None,
                uptime.group(1).strip() if uptime else None)

    def _parse_inventory(self, content: str) -> tuple:
        """解析设备型号和序列号"""
        model = re.search(r'PID:\s*(\S+)', content)
        serial = re.search(r'SN:\s*(\S+)', content)
        return (model.group(1) if model else None,
                serial.group(1) if serial else None)
```

### 5.2 异常分析器 analyzer.py

**建议位置：** `CoreBase/core/analyzer.py`

```python
"""
设备巡检异常检测模块
"""
from typing import Dict, List, Any


# 阈值配置
THRESHOLDS = {
    "cpu_5m": {"warning": 60, "critical": 80},
    "memory": {"warning": 80, "critical": 90},
    "uptime_days": {"warning": 1},  # 小于1天需关注
}


def analyze_temperature(temp: int, warning_limit: int, alarm_limit: int) -> str:
    """温度异常判断"""
    if temp >= alarm_limit:
        return "告警"
    elif temp >= warning_limit * 0.8:
        return "偏高"
    return "正常"


def analyze_cpu(cpu_5m: int) -> str:
    """CPU 异常判断"""
    if cpu_5m >= THRESHOLDS["cpu_5m"]["critical"]:
        return "严重"
    elif cpu_5m >= THRESHOLDS["cpu_5m"]["warning"]:
        return "警告"
    return "正常"


def analyze_memory(used_pct: float) -> str:
    """内存异常判断"""
    if used_pct >= THRESHOLDS["memory"]["critical"]:
        return "严重"
    elif used_pct >= THRESHOLDS["memory"]["warning"]:
        return "警告"
    return "正常"


def analyze_power(state: str) -> str:
    """电源状态判断"""
    if state.lower() in ("normal", "ok"):
        return "正常"
    return "故障"


def analyze_fan(state: str) -> str:
    """风扇状态判断"""
    if state.lower() in ("normal", "ok"):
        return "正常"
    return "故障"


def generate_device_conclusion(parsed_result) -> Dict[str, Any]:
    """生成设备整体评估结论"""
    anomalies = []

    for p in (parsed_result.power or []):
        if analyze_power(p["state"]) != "正常":
            anomalies.append(f"电源异常: {p['module']} 状态 {p['state']}")

    for f in (parsed_result.fan or []):
        if analyze_fan(f["state"]) != "正常":
            anomalies.append(f"风扇异常: {f['module']} 状态 {f['state']}")

    for t in (parsed_result.temperature or []):
        conclusion = analyze_temperature(t["temp"], t["warning"], t["alarm"])
        if conclusion != "正常":
            anomalies.append(f"温度{conclusion}: {t['sensor']} 当前{t['temp']}℃")

    for c in (parsed_result.cpu or []):
        cpu_5m = int(c["5m"].replace("%", ""))
        if analyze_cpu(cpu_5m) != "正常":
            anomalies.append(f"CPU {analyze_cpu(cpu_5m)}: Slot{c.get('slot', '?')} 5分钟{c['5m']}")

    for m in (parsed_result.memory or []):
        if analyze_memory(m["used_pct"]) != "正常":
            anomalies.append(f"内存{analyze_memory(m['used_pct'])}: Slot{m.get('slot', '?')} {m['used_pct']}%")

    if anomalies:
        return {"level": "异常", "anomalies": anomalies}
    return {"level": "正常", "anomalies": []}
```

```python
"""
巡检报告生成器
"""
from typing import List, Dict, Any


class InspectionReporter:
    """生成 Markdown 格式的巡检报告"""

    def __init__(self, title: str = "设备巡检结果"):
        self.title = title
        self.sections = []

    def add_section(self, title: str, content: str):
        self.sections.append((title, content))

    def generate_vendor_report(self, vendor: str, devices: List[Dict]) -> str:
        """按厂商生成巡检报告"""
        lines = [f"# {vendor} 设备巡检结果\n"]

        # 1. 设备系统信息
        lines.append("## 1、设备系统信息\n")
        lines.append("| IP | 设备名称 | 厂商 | 系统版本 |")
        lines.append("| --- | --- | --- | --- |")
        for d in devices:
            lines.append(f"| {d['ip']} | {d['name']} | {d['vendor']} | {d.get('version', 'N/A')} |")
        lines.append("")

        # 2. 电源状态
        lines.append("## 2、电源状态\n")
        lines.append("| 设备名称 | IP | 电源模块 | 状态 | 结论 |")
        lines.append("| --- | --- | --- | --- | --- |")
        for d in devices:
            for p in d.get("power", []):
                lines.append(f"| {d['name']} | {d['ip']} | {p['module']} | {p['state']} | {p.get('conclusion', '')} |")
        lines.append("")

        # ... 风扇、温度、CPU、内存等章节

        # 最终章节：整体评估
        lines.append("## N、设备整体信息分析评估\n")
        lines.append("| 设备名称 | 整体评估 | 分析结论 | 建议 |")
        lines.append("| --- | --- | --- | --- |")
        for d in devices:
            lines.append(f"| {d['name']} | {d.get('overall', 'N/A')} | {d.get('analysis', '')} | {d.get('suggestion', '')} |")

        return "\n".join(lines)

    def save_report(self, filepath: str, content: str):
        """保存报告到文件"""
        with open(filepath, "w", encoding="utf-8") as f:
            f.write(content)
```
