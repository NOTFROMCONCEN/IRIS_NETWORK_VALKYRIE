# 配置文件说明

## 配置文件列表

- `config.yaml` - 主配置文件（已提交到Git）
- `commands.yaml` - 巡检命令配置文件（支持按厂商和按主机名独立存储）
- `password.conf` - 密码配置文件（已添加到.gitignore，不会提交）
- `password_example.conf` - 密码配置示例文件

## 快速开始

### 1. 配置密码文件

复制示例密码文件并填入真实密码：

```bash
# Windows
copy password_example.conf password.conf

# Linux/Mac
cp password_example.conf password.conf
```

然后编辑 `password.conf`，填入真实的设备密码。

### 2. 配置设备列表

设备列表文件位于 `devices/devices.xlsx`，请参考 `devices/devices_example.xlsx` 示例文件创建。

设备表格式：

| 列名         | 必填 | 说明                                             |
| ------------ | ---- | ------------------------------------------------ |
| 设备名       | 是   | 设备名称，建议使用有意义的名称                   |
| IP地址       | 是   | 设备的IP地址                                     |
| 厂商         | 是   | 设备厂商：huawei, h3c, ruijie, maipu, wst, cisco |
| 端口         | 否   | SSH端口，默认22                                  |
| 设备型号     | 否   | 设备型号                                         |
| 位置         | 否   | 设备物理位置                                     |
| 联系人       | 否   | 设备联系人                                       |
| 备注         | 否   | 其他备注信息                                     |
| 跳板机地址   | 否   | 跳板机的IP地址（用于内网环境）                   |
| 跳板机端口   | 否   | 跳板机SSH端口，默认22                            |
| 跳板机用户名 | 否   | 登录跳板机的用户名                               |
| 跳板机密码   | 否   | 登录跳板机的密码                                 |

### 跳板机配置说明

本工具支持通过跳板机连接内网设备，适用于内网PC+跳板机的部署环境。

**使用场景：**
- 内网PC无法直接访问目标设备，需要通过跳板机中转
- 目标设备位于隔离网络，只能通过特定跳板机访问
- 需要通过堡垒机/跳板机进行统一访问控制

**配置方式：**

1. **设备清单配置（推荐）：** 在设备清单Excel中为每个设备单独配置跳板机信息
2. **全局配置：** 在 `config.yaml` 的 `network.jump_host` 部分配置全局默认跳板机

**连接流程：**
1. 程序首先连接到跳板机
2. 从跳板机发起SSH连接到目标设备
3. 使用 redispatch 将连接切换到目标设备类型
4. 正常执行巡检命令

**注意事项：**
- 设备清单中的跳板机配置优先级高于全局配置
- 建议在设备清单中配置跳板机，避免在配置文件中存储明文密码
- 跳板机用户名和密码为配置跳板机时的必填项

### 3. 修改主配置

编辑 `config.yaml` 和 `commands.yaml` 根据需要调整以下配置：

- 日志级别和输出目录
- 网络连接超时时间
- 巡检命令列表（建议在 `commands.yaml` 中维护）
- 特殊设备的主机名命令覆盖（在 `commands.yaml` 的 `hostnames` 下维护）
- 通知设置

## commands.yaml 规则

- `vendors` 用于维护厂商默认命令。
- `hostnames` 用于维护特殊设备命令，匹配 `devices.xlsx` 中的“设备名”字段，大小写不敏感。
- 当同一台设备同时命中厂商和主机名配置时，先加载厂商命令，再由主机名配置覆盖同名命令并追加额外命令。
- `hostnames` 条目中可选填写 `vendor`，用于限制该主机名覆盖只对指定厂商生效。

## 安全注意事项

- `password.conf` 文件包含敏感信息，已添加到 `.gitignore`，不会提交到Git
- 请妥善保管密码文件，不要将其分享给未授权人员
- 建议定期更换设备密码
