# 设备管理UI模块

## 概述

设备管理UI模块为网络设备巡检工具提供了一个基于Web的图形界面，用于管理设备Excel文件，并可直接从页面启动巡检任务。该模块使用Streamlit构建，可以通过`python main.py --ui`命令启动。

## 功能特性

- **设备查看**: 以表格形式查看所有设备信息
- **设备添加**: 通过表单添加新设备
- **设备编辑**: 修改现有设备信息
- **设备删除**: 删除指定设备
- **搜索过滤**: 按设备名、IP地址或厂商搜索设备
- **分组过滤**: 支持按设备分组筛选设备与巡检范围
- **统计分析**: 查看设备统计信息和数据质量检查
- **导入导出**: 支持CSV格式的数据导入导出
- **启动巡检**: 直接从UI发起巡检任务并查看实时日志
- **自动日志刷新**: 巡检运行时日志自动轮询刷新，页面保持非阻塞

## 安装要求

在运行UI模块前，需要安装额外的依赖：

```bash
pip install streamlit>=1.28.0
```

或者通过项目的requirements.txt安装：

```bash
pip install -r requirements.txt
```

## 使用方法

### 启动UI界面

```bash
# 在仓库根目录启动设备管理UI
python main.py --ui
```

启动后，控制台会显示访问地址（通常是 http://localhost:8501），在浏览器中打开该地址即可使用。

### 命令行参数

- `--ui`: 启动设备管理Web界面

### 界面说明

UI界面分为以下几个主要部分：

1. **侧边栏**: 包含操作菜单、搜索过滤和文件操作
2. **主区域**: 根据选择的操作显示相应内容
3. **统计面板**: 显示设备总数、厂商数量等统计信息

#### 可用操作

1. **📋 查看设备**: 以可编辑表格形式显示所有设备，支持批量修改
2. **➕ 添加设备**: 通过表单添加新设备，支持必填字段验证
3. **✏️ 编辑设备**: 选择特定设备进行编辑
4. **🗑️ 删除设备**: 选择并删除设备（需要确认）
5. **📈 统计分析**: 查看设备统计信息和数据质量检查
6. **📦 导入导出**: 通过CSV导入或导出设备清单
7. **🚀 启动巡检**: 从Web界面启动巡检、停止任务并查看自动刷新的运行日志，支持厂商、分组、IP、日志模式、模拟运行、自定义配置文件和自定义结果目录

## 设备字段说明

| 字段名   | 必填 | 说明               | 示例               |
| -------- | ---- | ------------------ | ------------------ |
| 设备名   | 是   | 设备的唯一标识名称 | SD91SN0A-N1        |
| IP地址   | 是   | 设备的IP地址       | 192.168.1.1        |
| 生产厂商 | 是   | 设备厂商           | 华为、H3C、CISCO等 |
| 端口     | 否   | SSH端口，默认22    | 22                 |
| 备注     | 否   | 设备备注信息       | 核心交换机         |

## 数据验证

UI模块会对输入的数据进行验证：

1. **必填字段检查**: 设备名、IP地址、生产厂商不能为空
2. **IP地址格式**: 必须符合IPv4地址格式
3. **端口范围**: 端口号必须在1-65535之间
4. **设备名唯一性**: 设备名不能重复
5. **IP地址唯一性**: 警告重复的IP地址

## 与最新巡检能力的对齐

- 巡检页面已支持 `--vendor`、`--group`、`--ip`、`--logmode`、`--dry-run`、`--config`、`--output-dir` 对应的 Web 参数。
- 厂商下拉会优先展示程序当前支持的厂商关键字，同时兼容设备清单里已有的历史厂商值。
- 分组筛选规则与 CLI 保持一致，支持 `分组_设备名`、`[分组]设备名`、`分组-设备名` 三种命名格式。
- 自定义配置文件和结果目录支持相对 CoreBase 的路径或绝对路径；配置文件缺失时主程序会回退到默认配置继续执行。

## 文件备份

每次保存设备数据时，系统会自动创建备份文件，保存在`devices/backups/`目录下，文件名格式为`devices_backup_YYYYMMDD_HHMMSS.xlsx`。

## 开发说明

### 模块结构

```
ui/
├── __init__.py          # 模块初始化和UI启动函数
├── device_manager.py    # 设备管理器核心类（CRUD操作）
├── app.py               # Streamlit应用主界面
└── README.md            # 本文档
```

### 核心类：DeviceManager

`DeviceManager`类提供了设备Excel文件的所有CRUD操作：

```python
from ui.device_manager import DeviceManager

# 初始化
manager = DeviceManager("devices/devices.xlsx")

# 加载设备
devices = manager.load_devices()

# 添加设备
new_device = {
    "设备名": "测试设备",
    "IP地址": "192.168.1.100",
    "生产厂商": "测试厂商",
    "端口": "22",
    "备注": "测试设备"
}
manager.add_device(new_device)

# 搜索设备
results = manager.search_devices(keyword="SD", vendor="CISCO")

# 获取统计信息
stats = manager.get_statistics()
```

### 扩展功能

如需扩展UI功能，可以：

1. **添加新字段**: 优先在`core/device_inventory.py`中扩展统一设备模型，再在`device_manager.py`和`app.py`中接入
2. **添加新操作**: 在`app.py`中添加新的操作页面
3. **修改界面样式**: 修改Streamlit组件的布局和样式
4. **添加数据验证**: 在`validate_device()`方法中添加更复杂的验证逻辑

## 故障排除

### 常见问题

1. **无法启动UI**: 确保已安装streamlit，且版本>=1.28.0
2. **导入错误**: 检查Python路径，确保可以从项目根目录导入模块
3. **文件权限错误**: 确保对`devices/`目录有读写权限
4. **Excel文件格式错误**: 确保Excel文件包含必需的列（设备名、IP地址、生产厂商）

### 错误处理

- 文件操作错误会自动记录到控制台
- 数据验证错误会在UI界面显示错误信息
- 网络错误（如无法访问Streamlit服务器）会显示相应的错误信息
